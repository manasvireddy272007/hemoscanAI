# HemoScan AI — Backend API v2
### FastAPI + Groq LLaMA3 AI | Anemia Detection

---

## 🏗️ Architecture

```
Browser (Frontend)
      │
      │  POST /analyze  (JSON — no API key)
      ▼
 FastAPI Backend  ◄── GROQ_API_KEY (environment variable, secret)
      │
      │  Groq API call (server-to-server, key never exposed)
      ▼
 Groq LLaMA3-70B
      │
      ▼
 Structured JSON Response → Frontend
```

**The API key NEVER touches the frontend. It lives only on your server.**

## ⚙️ Setup & Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Set your API key (pick one method)

**Method A — .env file (recommended for local dev):**
```
# .env
GROQ_API_KEY=gsk_your_key_here
```

**Method B — Export in terminal:**
```bash
# Mac/Linux
export GROQ_API_KEY="gsk_your_key_here"

# Windows CMD
set GROQ_API_KEY=gsk_your_key_here

# Windows PowerShell
$env:GROQ_API_KEY="gsk_your_key_here"
```

### 3. Start the server
```bash
uvicorn main:app --reload --port 8000
```

## 🌐 Frontend Integration

In your existing `analyze()` JS function, replace the local logic with:

```javascript
async function callBackend(payload) {
  try {
    const response = await fetch('http://localhost:8000/analyze', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload)
    });

    if (!response.ok) {
      const err = await response.json();
      throw new Error(err.detail || 'Analysis failed');
    }

    return await response.json();
  } catch (error) {
    console.error('Backend error:', error);
    throw error;
  }
}
```

## 🔒 Security Notes

| ✅ DO                                         | ❌ DON'T                              |
|-----------------------------------------------|---------------------------------------|
| Store key in `.env` file                      | Put key in frontend JS               |
| Add `.env` to `.gitignore`                    | Commit `.env` to GitHub              |
| Use environment variables on cloud deployment | Hardcode key in `main.py`            |
| Restrict CORS to your domain in production   | Use `allow_origins=["*"]` in production |

### Cloud Deployment (Render / Railway / Heroku)
Set environment variables in the dashboard — **not** in code:
```
Key:   GROQ_API_KEY
Value: gsk_your_actual_key
```

## 🤖 How the AI Works

1. **Rule engine** scores the CBC data (0–100) using WHO thresholds
2. The score + all patient parameters are sent to **Groq LLaMA3-70B**
3. The AI returns a structured JSON with clinical explanation + diet plan
4. Backend validates and returns the response to your frontend

The rule engine ensures the AI has a solid base — the LLM adds nuanced clinical interpretation and personalized dietary advice.
